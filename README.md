# Museum AI

Бэкенд и мобильное приложение для музейного гида. Пользователь фотографирует картину — приложение распознаёт её, дает возможность выделить границы на экране и возвращает историю от лица художника, аудиорассказ и фото художника.

## Как это работает

1. Пользователь делает снимок картины в мобильном приложении.
2. Пользователь настраивает рамку выделения пальцем, обрезая лишний фон.
3. Мобильное приложение отправляет обрезанное фото на `POST /experience`.
4. Сервер сравнивает фото с эталонными изображениями.
5. В ответ приходят: данные о картине, рассказ от лица художника, ссылки на аудио и фото художника.

## MVP

Поддерживаются две картины:
- **Mona Lisa** — Леонардо да Винчи, 1503
- **The Starry Night** — Винсент ван Гог, 1889

## Технологии

### Бэкенд
- **Python 3.12**
- **FastAPI** — веб-фреймворк
- **uvicorn** — сервер
- **Pillow + imagehash** — распознавание картин по фото
- **gTTS** — генерация аудио (Google Text-to-Speech)

### Фронтенд
- **React Native (Expo SDK 54)** — мобильное приложение
- **TypeScript** — язык разработки
- **React Navigation v7** — навигация
- **expo-camera** — работа с камерой
- **expo-image-manipulator** — обрезка изображений на клиенте
- **expo-av** — воспроизведение аудио

## Запуск

### Запуск бэкенда
```bash
# Перейти в папку бэкенда
cd backend

# Активировать виртуальное окружение
source venv/bin/activate

# Установить зависимости
pip install -r requirements.txt

# Запустить сервер
uvicorn main:app --reload
```
Сервер запустится на `http://127.0.0.1:8000`
Документация API доступна на `http://127.0.0.1:8000/docs`

### Запуск фронтенда
Перед запуском пропишите IP-адрес вашего компьютера (из сети Wi-Fi) в переменной `BASE_URL` в файле `frontend/services/api.ts`, чтобы телефон мог подключиться к локальному серверу.

```bash
# Перейти в папку фронтенда
cd frontend

# Установить npm-пакеты
npm install --legacy-peer-deps

# Запустить сборщик
npm start
```
Отсканируйте появившийся QR-код через приложение **Expo Go** на телефоне (версия 54.0.8). 

Если вы тестируете удаленно (компьютер и телефон в разных сетях), запускайте сборщик через тоннель:
```bash
npx expo start --tunnel
```

*Примечание: Чтобы тестировать интерфейс без запущенного бэкенда, просто нажмите на карточки картин внизу главного экрана приложения. Они откроются с использованием локальных оффлайн-файлов из папки `frontend/assets`.*

## API

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/` | Проверка что сервер работает |
| POST | `/recognize` | Распознать картину по фото |
| GET | `/painting/{id}` | Данные о картине |
| GET | `/story/{id}` | Рассказ художника |
| POST | `/speech/{id}` | Сгенерировать аудио |
| GET | `/audio/{id}` | Получить аудиофайл |
| GET | `/artist/{id}` | Фото художника |
| POST | `/experience` | Всё сразу: распознавание + история + аудио |

## Добавление новой картины

1. Добавить эталонное фото в `backend/assets/paintings/{id}.jpg`
2. Добавить фото художника в `backend/assets/artists/{artist_id}.jpg`
3. Добавить запись в `backend/database/paintings.json`
4. В `backend/services/image_matcher.py` добавить `{id}` в словарь `PAINTINGS`
5. *(Опционально)* Для локальных тестов на фронтенде скопировать файлы художника `.jpg` и озвучки `.mp3` в `frontend/assets/` и прописать их в `MOCK_RESULTS` в `frontend/screens/HomeScreen.tsx`.
