# museum-ai

## О проекте
Приложение для музеев на FastAPI. Пользователь фотографирует картину,
приложение распознаёт её и рассказывает историю от лица художника.
MVP ограничен двумя картинами: Mona Lisa и Starry Night.

Разработчик — начинающий. Важно: объяснять происходящее, не перепрыгивать
через этапы, не предлагать сложную архитектуру раньше времени.

## Структура проекта
museum-ai/

backend/

├── main.py              — точка входа, запуск FastAPI-сервера

├── requirements.txt     — список библиотек проекта

├── routes/

│   ├── recognize.py     — POST /recognize, принимает фото, передаёт в image_matcher

│   ├── stories.py       — рассказы художников

│   └── avatars.py       — изображения художников

├── services/

│   ├── image_matcher.py    — сравнение фото с эталонами

│   ├── story_generator.py  — текст рассказа

│   └── speech_generator.py — текст в аудио

├── database/

│   └── paintings.json   — простая "база" в виде JSON

└── assets/

├── paintings/        — эталонные фото (.jpg)

├── artists/          — фото художников (.jpg)

└── audio/            — сгенерированные mp3

## Принцип разделения
- **routes** — "дверь": принимает запрос, сам не думает
- **services** — "комната внутри": реальная логика

## План этапов (MVP)
1. FastAPI запущен, GET / → {"status": "working"}  ✅ готово
2. POST /recognize — принимает изображение (пока просто подтверждение)  ← мы здесь
3. paintings.json с данными о картинах
4. Распознавание двух картин (сравнение с эталонами)
5. GET /painting/{id}
6. GET /story/{id} — текст из JSON, без LLM
7. POST /speech/{id} — генерация речи
8. GET /artist/{id}
9. POST /experience — всё сразу
